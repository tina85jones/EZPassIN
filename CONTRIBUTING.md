# Contribution Guidelines

We welcome contributions to the "EZ Pass IN" project! By contributing, you help make the project better for everyone. Please follow the guidelines below to ensure smooth collaboration.

## How to Contribute

We appreciate contributions of any kind, whether it's a bug fix, a feature request, or a general improvement. Here's how to contribute:

1. **Fork the repository** to your own GitHub account.
2. **Create a new branch** for your changes. 
   - Use a descriptive name for the branch.
   - Example: `feature/add-expense-form` or `bugfix/fix-header`.
3. **Make your changes** to the codebase, ensuring that your changes are well-documented and follow the existing code style.
4. **Commit your changes** with a detailed commit message explaining the reason for the change.
5. **Push the changes** to your forked repository.
6. **Create a Pull Request (PR)** to merge your changes into the `main` branch.

## Reporting Issues

To report a bug or an issue, follow these steps:

1. Check if the issue has already been reported by searching the repository's open issues.
2. If not, create a new issue by clicking the "New Issue" button.
3. Provide clear, concise information about the bug or issue, including steps to reproduce, expected behavior, and actual behavior.

## Suggesting Features

Feature requests are always welcome! If you have a feature in mind that would improve the project, please follow these steps:

1. Check if the feature has already been suggested by searching the open issues.
2. If not, open a new issue and describe the feature in detail.
3. Provide any relevant use cases, user stories, or mockups if applicable.

## Code of Conduct

We ask all contributors to adhere to our code of conduct, which ensures a respectful and welcoming environment for everyone. Please read our [Code of Conduct](./CODE_OF_CONDUCT.md) for more details.

## Review Process

Once you submit your Pull Request (PR), it will be reviewed by the project maintainers. The review process generally involves the following steps:

1. The PR will be reviewed for quality, coding standards, and compliance with project guidelines.
2. If any changes are necessary, feedback will be provided, and you can update the PR accordingly.
3. Once everything looks good, your PR will be merged into the `main` branch.

Thank you for your contributions!
